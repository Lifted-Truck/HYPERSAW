Reading: iFFT is ~3× faster, but both have order-of-magnitude headroom even derated 4× for min-spec — so throughput doesn't decide. What decides: the L0-1 parity oracle (ε = 1e-6 vs the JS per-sample accumulators — structurally impossible through a windowed-OLA pipeline) and control-rate semantics (bank honors the 16-sample tick; iFFT quantizes control to a ~23 ms hop, smearing fast K gestures).

Proposed decisions — for your ratification

ADR-018 — Renderer: oscillator bank · proposed

Table + linear interp per SPEC §6.6. iFFT recorded as the escape hatch (revisit trigger: > ~1k sines per note, or min-spec E-6 measurements falling short).

ADR-019 — GUI stack: embedded webview (choc) · proposed

Reuses the prototypes' canvas/JS/CSS nearly verbatim — the strongest possible fit to ADR-013's "match the prototype design language." Precedents: JUCE 8's WebView UI is built on choc; CLAP has a draft webview extension (1.2.7+); Cmajor's CLAP exporter defaults to webview.

Risks with owners: keyboard-focus stealing in Live (known pattern → key-forwarding from day one); webview-through-clap-wrapper unconfirmed in the wild (→ smoke test before GUI work); multi-instance memory (→ measure with 8 instances in Phase 2). Fallback: iPlug2 IGraphics or Dear ImGui, at the cost of full visual reimplementation.

E-6 hardware envelope · proposed

Min-spec: Apple M1 base / 4-core 2018-class Intel ultrabook; Windows x64 AVX2. 44.1 kHz @ 128-sample buffer. The E-6 patch (8-voice poly, M = 32 SAW) must hold < 50% of one core on min-spec.
