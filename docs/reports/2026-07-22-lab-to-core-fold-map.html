Recommended fold sequence

  1. Supersets first — tone tilt/HPF/hi-tame, drift modes + centre-pin, freq smoothing, keep-phase, pan-motion, comp/limiter. Each: reference edit → parity holds (default bit-exact) → one increment. Lowest risk, biggest surface.
  2. New detune laws — harmonic+reach, stretch, golden, spread/anchor. Each is a reference law-table addition → new golden → ADR. Inert when unselected, but each is a spec change to swarmsaw.html.
  3. Divergences needing a decision — root-pivot topology (keep both; ADR), pan default image (ADR — it changes existing patches' stereo), saw-shape retarget (supersede ADR-058; needs measured synth-saw profiles for the base bank).
  4. New modules — polyphonic KS comb (as an FX-rack slot, reusing the filter/notch/time-core precedent).
  5. Face & naming — the visual language + name-by-feel glossary (K→"cooperation", swarm→"horde"), Phase 5.
Parity contract stays absolute: every superset must render bit-identically at its default so the existing goldens pass; every divergence gets its own re-measured golden + ADR. The lab file itself is not ported — it is the spec-in-code the reference edits are transcribed from.
